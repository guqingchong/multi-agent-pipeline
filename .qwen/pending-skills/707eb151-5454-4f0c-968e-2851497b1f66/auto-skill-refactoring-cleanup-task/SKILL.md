# 复杂代码库重构清理任务

## 概述
这是一种复杂的重构清理任务，涉及合并重复定义、删除旧常量引用、更新测试并确保功能完整性。适用于需要从多个模块迁移到统一注册表的大型重构项目。

## 适用场景
- 需要将多个模块中的重复常量定义合并到单一注册表
- 需要删除废弃的配置文件并将其功能整合到现有模块
- 需要更新引用旧常量的测试文件以使用新的注册表API
- 需要在重构后验证功能完整性

## 操作步骤

### 1. 识别重复定义
- 使用 `grep_search` 找出所有重复的常量定义
- 确定真正的"单一事实来源"（如 `REGISTRY`）

### 2. 逐步迁移定义
- 从源模块开始，逐步修改引用点
- 确保向后兼容性（如 `models.py` 中的 `PHASE_NAMES` 需要保持索引对应关系）
- 为新旧方式建立桥接（如导入 `REGISTRY` 并基于其数据动态生成）

### 3. 删除废弃文件
- 确认文件不再被直接使用
- 将其功能迁移到合适的模块
- 更新所有引用该文件的地方

### 4. 更新测试文件
- 修改导入语句以使用新的API
- 更新测试逻辑以适应新的实现方式
- 保持测试覆盖率不变

### 5. 验证功能完整性
- 运行受影响模块的单元测试
- 运行集成测试以确保整体功能正常
- 修复因重构引入的任何问题

## 注意事项
- 当处理有向后兼容性要求的定义时，需要小心维护API契约
- 重构过程中可能需要临时保留一些旧代码直到所有引用都被更新
- 修复测试时要确保测试仍然验证正确的功能行为
- 在进行重大重构前，始终备份代码或使用版本控制
